# META CAROUSEL: Platform-Native Content Knowledge
## Platform: Instagram/Facebook | Format: Carousel | Last Updated: April 2026

---

## DRAFT FIELDS FOR AI GENERATION

When generating content for a Meta Carousel, the AI must produce these specific fields:

1. **Cover Slide (Frame 1)**: The hook slide. Must present an incomplete thought that demands a swipe. Bold typography. Never reveal the full answer on the cover. Maximum 8-10 words as headline. Think of it as a book cover, not a title slide.

2. **Slides (Array of middle frames, typically 5-8)**: One idea per slide. Each slide has a heading and body text. Maximum 30 words per slide. Content should create a progressive reveal where each slide builds on the last. The last line of one slide can lead into the next to create a narrative thread.

3. **Closing Slide (Final frame)**: CTA slide. Visually distinct from middle slides (different background color or simplified layout). Single clear CTA. Brand logo subtle, not dominant. Options: "Save this for later" / "Share with someone who needs this" / "Follow @handle for more" / "Comment your experience"

4. **Caption**: Post caption. 100-200 words. The caption complements the carousel, it does not repeat it. Structure: hook line, context or "why" behind the carousel's topic, CTA. 3-5 hashtags at the end.

5. **Hashtags**: 3-5 total. Mix of volume tiers.

6. **Design Direction**: Visual design guidance for the entire carousel. Color palette reference from brand assets, font hierarchy (heading font + body font), layout pattern (centered, left-aligned, asymmetric), consistency rules that must be maintained across all slides.

## RESEARCH-BACKED RULES

### Why Carousels Work in 2026
- Carousels generate higher save rates than any other Instagram format because they offer reference value people want to return to
- Each swipe counts as an interaction, increasing dwell time and signaling deep engagement to the algorithm
- Carousel renaissance in 2026: multi-slide educational posts are experiencing the highest engagement rates since the format launched
- Even incomplete swipes (partial swipe and return) count as interaction signals
- Carousels outperform static posts significantly for engagement metrics across all account sizes

### Slide Structure Rules
- Frame 1 (Cover): This is a book cover, not a title slide. It must create a curiosity gap that can only be resolved by swiping. Examples of strong covers: "The 5 things killing your engagement" or "What nobody tells you about pricing." Examples of weak covers: "5 Tips for Better Engagement" or "A Guide to Pricing" (these reveal too much and remove the reason to swipe)
- Frames 2-8 (Body): ONE point per frame, no exceptions. Provide enough whitespace for mobile readability. All text should be scannable in 2-3 seconds per frame. Use the last line of one frame as a setup for the next frame to create a narrative pull
- Final Frame: Clear CTA that is visually distinct from the middle frames. Use a different background color or a simplified layout that clearly signals "this is the end, here's what to do next"

### Design Rules
- Consistent visual language across ALL frames: same font family, same color system, same layout grid
- Minimum 24pt equivalent for headline text, 16pt equivalent for body text (for mobile readability)
- No more than 2 fonts per carousel
- High contrast between text and background; test readability by imagining viewing on a phone screen in daylight
- Frame count sweet spot: 7-10 frames for educational content, 5-7 for storytelling, 3-5 for before/after comparisons
- Avoid clutter; every element on the slide should earn its place

### Caption Strategy for Carousels
- The caption complements the carousel; it should not repeat the content inside the slides
- Use the caption to provide the "why" or personal context behind the carousel's "what"
- Save-oriented CTAs work best for educational carousels because saves are the strongest algorithmic signal for this format
- A question-based CTA drives comments: "Which of these resonated most? Tell us in the comments"

### Content Types That Work Best as Carousels
- Step-by-step tutorials and how-to guides
- Listicles with depth ("7 mistakes brands make with..." with one mistake per slide)
- Before/after comparisons
- Data breakdowns and statistics with visual presentation
- Framework explanations (one component per slide)
- Myth-busting ("What people think vs what's actually true")
- Product feature breakdowns
- Tips and checklists that people save for later reference

### Static vs Carousel Decision
- Use a static when you have a single idea, want emotional impact, or the content is for quick consumption
- Use a carousel when you have multiple ideas, want educational depth, or the content has reference value people will return to
- When in doubt, a carousel almost always performs better for engagement metrics than a static
